Python Emailing Package

Problem definition

During my tenure at Ehealth4Everyone, I identified a recurring task: the need to send automated emails for various projects. Each project required incorporating email functionality into the codebase, leading to redundant efforts. Recognizing the potential for efficiency improvement, I decided to streamline this operatoin through the development of a Python Emailing Package.

Despite exploring existing packages, none offered the level of robustness necessary to fulfill all our emailing requirements. Motivated by this gap in the market, I undertook the challenge of developing a highly capable and adaptable Python Emailing Package. This package does not only provide a solution for Ehealth4Everyone's challenges but its robustness also ensures it caters to the needs of any Python developer looking to send emails efficiently and effectively.


I worked remotely with 2 other contributors. My role in the team was to design and implement the Python Emailing Package as well as write unit tests and perform the intial CI/CD for the package.

Solution

A Python package capable of extracting email contents to be sent to multiple recipients was developed. Some key features of the package include:

  • Checks and cleans common erros in email addresses: The package checks for and rectifies common mistakes such as unexpected spaces, fullstops, etc., in the recipients' email addresses.
  • Checks for valid email address syntax: Using regex pattern matching, the package checks for valid email address syntax and eliminates the faulty ones.
  • Extracts content to be sent: Unlike other emailing packages, this package can extract the message to be sent from various sources such as .html and .txt files, or directly from a string.
  • Reads recipient data from various files: To accommodate sending emails to multiple recipients, the package is capable of automatically detecting and reading email addresses from a variety of file formats and data structures including lists, tuples, sets, dictionaries, .json files, and .csv files.
  • Includes recipient's name in the email message: If the names of the recipients are provided, the package can personalize emails by including the recipient's name after a specified introductory word (e.g., Hello ).
  • Auto detects the SMTP host of the provided sender's email address
  • Attaching and displaying images in email: The package can attach a file, or multiple files from a specified directory in an email. Furthermore, it can display provided images as part of the email message, enhancing the visual appeal of the email. This feature sets it apart from other Python packages.
